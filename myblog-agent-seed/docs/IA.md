# 전체 구조 개요

> 단순 “글 목록”이 아니라 랜딩 페이지처럼 구성하고, 아래 2가지 목적을 동시에 해결한다.

## 소개 목적: “이 블로그/작성자가 누구인지”

## 탐색 목적: “어떤 기술 글이 있는지 빠르게 찾기”

## 섹션 우선순위 (권장 순서)

### S0. 글로벌 헤더 (기존 유지 + 강화)

- 목적:
  - 빠른 이동 / 브랜드 인지

- 구성
  - 로고(해학)
  - 카테고리 / 검색 / 다크모드 / 메뉴
  - (모바일) 햄버거 + 캡슐 메뉴
  - 비고: 기존 헤더 유지 가능, 스타일만 개선

### S1. Hero (소개 + 핵심 가치 제안)

- 목적:
  - 첫 화면에서 블로그 성격/작성자 강점 전달

- 구성
  - 한 줄 소개 (예: 백엔드 개발자 / 구조적 문제 해결 / 기록 중심)
  - 짧은 설명 (무엇을 쓰는 블로그인지)
  - 주요 CTA 3개
  - 대표 글 보기
  - 전체 카테고리 보기
  - GitHub
  - 추천 요소
  - 기술 태그 pill (ComputerScience / Develop / Engineering / TroubleShooting)
  - 최근 작성 주제 6개

### S2. Quick Links (빠른 진입 허브)
- 목적: 소개에서 바로 행동으로 연결
- 구성 (카드/칩 형태)
  - 대표 카테고리 바로가기
  - GitHub
  - 포트폴리오/이력(있다면)
  - 구독 / 방명록
  - 비고: 모바일에서 가로 스크롤 카드 가능

### S3. Featured Content (대표 글 / 시리즈)

- 목적:
  - 글 탐색 진입점 강화

- 구성
  - 대표 글 3개 또는 시리즈 2개 + 글 4개
  - 각 카드에 카테고리/핵심 요약/난이도/작성일 표시(선택)

- 추천 타입
  - “입문용”
  - “실무/문제해결”
  - “구조/설계”

### S4. About & Tech Snapshot (소개 + 기술 스택 요약)
- 목적: 포트폴리오형 신뢰감 부여
  - 구성
  - 짧은 자기소개
  - 핵심 강점 3개 (예: 구조 설계 / 성능 개선 / 문서화)
  - 기술 스택 그룹
    (ComputerScience / Develop / Engineering / TroubleShooting)

- 비고:
  - 너무 길지 않게, 상세 소개는 별도 링크 가능

### S5. Category Hub (기술 블로그 허브 핵심)
- 목적:
  - 카테고리 중심 탐색
- 구성
  - 상위 카테고리 카드 (예: CS / Development / Engineering / TroubleShooting)
  - 각 카드에 하위 카테고리 미리보기
  - 카테고리별 글 수/설명
  - UX 포인트
  - 모바일: 탭 또는 horizontal card
  - PC: 2~4열 카드 그리드

### S6. Recent Posts / Updates (최신 글)

- 목적:
  - “살아있는 블로그” 신호

- 구성

  - 최신 글 리스트 5~10개
  - 카테고리 + 제목 + 날짜

- 비고:
  - 기존 티스토리 최신 글 블록과 연결 가능

### S7. Subscribe / Guestbook / Contact (관계 형성)

- 목적:
  - 재방문/구독/소통 유도
- 구성
  - 구독 안내 카드
  - 방명록 이동
  - 이메일/연락 링크

- 비고:
  - 현재 있는 요소를 디자인 재배치

### S8. Footer (브랜드/링크/정리)
- 목적:
  - 마무리 정보 제공

- 구성
  - 블로그명, 짧은 문구
  - 주요 링크
  - - 저작권/운영 안내
