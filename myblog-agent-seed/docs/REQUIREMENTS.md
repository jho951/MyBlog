# 요구사항

## 범위

- 포함:
  - 메인 페이지 IA(정보구조) 재설계
  - “소개(Who I am) + 기술 블로그 허브(What I write)” 이중 목적 구조로 개편
  - 상단 Hero 섹션에서 블로그 정체성/강점/핵심 카테고리 요약 제공
  - 포트폴리오형 소개 섹션 추가
  - 자기소개(한 줄 소개 + 상세 소개)
  - 기술 스택 요약(Strong / Familiar / Learning 등 단계 구분)
  - 주요 프로젝트/대표 글/깃허브 링크 노출
  - 기술 블로그 허브 섹션 강화
  - 카테고리별 진입 카드(예: CS/BE/FE/INFRA)
  - 최신 글 / 추천 글 / 시리즈 글 영역 구성
  - “전체 글 보기” 및 카테고리 이동 동선 최적화
  - 기존 티스토리 기능과의 공존
  - 티스토리 글 목록/카테고리/검색/구독/방명록 흐름 유지
  - 티스토리 스킨 커스터마이징 범위 내에서 UI 개선
  - 브랜딩/신뢰 요소 강화
  - GitHub/연락처/소개 문구 정리
  - 방문자용 신뢰 요소(운영 목적, 글 작성 기준, 업데이트 방식) 노출
  - 반응형 UI 개선
  - 모바일 우선 레이아웃 구성
  - 태블릿/데스크톱에서 소개 섹션 + 콘텐츠 허브 가독성 개선
  - 성능/가독성 중심 개선
  - 이미지/스크립트 최소화
  - 초기 렌더에서 핵심 정보 우선 노출
- 제외:
  - 티스토리 플랫폼 외부로의 완전한 사이트 이전(예: Next.js 독립 배포로 전환)
  - 백엔드/API 신규 구축
  - 회원 기능/커스텀 DB/로그인 시스템 구현
  - 기존 전체 포스트 콘텐츠의 대량 리라이트
  - 카테고리 체계 전체 재분류(필요 시 후속 작업으로 분리)
  - SEO 고도화 전체(구조화 데이터/검색엔진 운영 전략 전체 수립 등)는 1차 범위 제외
  - 포트폴리오 전용 상세 프로젝트 페이지 신규 구축(링크 허브 수준까지만)

## 가정

- 명시된 가정:
  - 대상 페이지는 티스토리 블로그 메인 페이지이며, 목표는 “개인 소개 + 기술 블로그” 형태의 랜딩/허브 페이지로 개선하는 것이다.
  - 참고 레퍼런스는 wnghdcjfe.github.io의 **포트폴리오형 섹션 구성 방식(소개, 프로젝트, 기술, 수상, 블로그 링크)**이며, 이를 티스토리 환경에 맞게 재해석한다.
  - 현재 블로그에는 이미 소개성 문구, 카테고리 진입, 구독, 방명록, 전체 글 목록 요소가 존재하며, 이를 완전히 제거하지 않고 재배치/강화한다.
- 도출된 가정:
  - 티스토리 스킨 구조/치환자 제약으로 인해 완전 자유로운 SPA 구조는 어렵고, HTML/CSS/JS 커스터마이징 중심으로 구현한다. (도출된 가정)
  - 메인 페이지에서 가장 중요한 KPI는 “첫인상(소개)”와 “콘텐츠 탐색 전환(카테고리/글 클릭)”이다. (도출된 가정)
  - 현재 블로그의 강점은 카테고리 볼륨과 기술 기록량이므로, 메인 페이지는 ‘많이 쓴 사람’이라는 신뢰감을 빠르게 전달해야 한다. 현재 카테고리/전체 글 수가 노출되어 있음.
  - 기존 방문자 중 일부는 “글 탐색 목적”, 일부는 “작성자/포트폴리오 확인 목적”일 가능성이 높아, 두 목적을 동시에 만족하는 IA가 필요하다. (도출된 가정)

## 비기능 요구사항

- 신뢰성:
  - 티스토리 기본 기능(글 목록 이동, 카테고리 이동, 구독, 방명록 링크)이 깨지지 않아야 한다.
  - 메인 페이지 커스터마이징 적용 후에도 JS 오류로 핵심 네비게이션이 동작 불능 상태가 되지 않아야 한다.
  - 외부 링크(GitHub/연락처 등) 클릭 실패를 줄이기 위해 링크 검증 가능한 구조로 관리한다.
  - 티스토리 스킨 업데이트/수정 시에도 섹션 단위로 유지보수 가능하도록 모듈형 CSS/JS 구조를 유지한다.
- 보안:
  - 사용자 입력을 직접 받는 커스텀 폼(이메일 수집 등)은 1차 범위에서 구현하지 않거나, 구현 시 XSS/스팸 대응 가능한 티스토리 기본 기능 또는 외부 검증된 서비스만 사용한다.
  - 외부 스크립트 사용을 최소화하고, 출처가 불분명한 JS 위젯 삽입을 지양한다.
  - 링크/배너/스크립트 삽입 시 관리자만 수정 가능한 정적 설정 중심으로 운영한다.
  - 개인정보(이메일/연락처) 노출 시 수집 목적 없는 단순 표시 원칙을 따른다.
- 성능:
  - 메인 페이지 첫 화면(Hero + 핵심 CTA + 주요 카테고리)은 스크롤 없이 핵심 정보 인지 가능해야 한다.
  - 초기 로딩 시 불필요한 애니메이션/대형 이미지 사용을 줄이고, 텍스트/카드 중심 구조를 우선한다.
  - 모바일 환경에서 스크롤 끊김 없이 탐색 가능해야 하며, 과도한 JS 연산(슬라이더/복잡한 인터랙션 다중 실행)은 지양한다.
  - 이미지 사용 시 카테고리 카드/썸네일은 지연 로드 또는 경량화된 리소스를 우선 적용한다.
  - 광고/구독/방명록/카테고리/글 목록이 함께 있는 현재 구조 특성을 고려해, 레이아웃 시프트(콘텐츠 밀림)를 최소화하는 배치 전략을 사용한다. (현재 메인에 구독/방명록/전체 글이 함께 노출됨)

## 구현 메모 (2026-02-25)

- 1차 구현 대상:
  - `src/pages/main.html`에 요구사항 IA 순서 기반 랜딩 허브 섹션(Hero / Quick Links / Featured / About&Tech / Recent / Connect) 추가
  - `src/style/style.css`에 홈 전용 스타일 추가(모바일 우선)
  - 바닐라 JS로 카드/탭 렌더링 및 최신 글 목록 재구성(`#ttLatestTrack`를 데이터 소스로 사용)
- 기존 티스토리 기능 공존 전략:
  - 카테고리 허브(`catHub`, `catStack`)와 구독 CTA(`ttSubscribePromo`)는 유지
  - 기존 장식성 섹션(인용/랜덤/방명록 카드/연락 카드 등)은 홈 화면에서 숨김 처리
  - 숨긴 최신글 슬라이더는 DOM 상 유지하여 최신 글 데이터 소스로 재활용
- 후속 작업(권장):
  - 대표 콘텐츠/빠른 링크 데이터의 실제 운영 링크로 정교화
  - 카테고리/글 수 자동 반영(티스토리 치환자 활용 가능 범위 검토)
  - 홈 랜딩 A/B 문구 및 CTA 클릭 흐름 점검
  - `theme-craft` 테마 사용 시 홈 섹션 래퍼 배경(영역 박스 느낌)과 개별 카드(`rp4-card`)의 시각 계층을 분리해, 섹션 배경 대신 카드 입체감으로 정보 구획을 표현한다.
  - `rp4` 영역은 카드 박스 장식보다 아이콘 오브젝트(`rp4-icon`)의 3D 스타일을 강조해 시선을 유도한다.
  - 카테고리 배너(`catHub`, `catStack`)는 카드 전체 클릭 구조를 유지하고 `바로가기` 보조 버튼 텍스트는 제거해 시각 복잡도를 줄인다.
  - 카테고리 배너 CTA 제거 후 텍스트 간격(`kicker/title/desc`)과 내부 패딩을 재조정해 빈 공간이 남지 않도록 한다.
  - 카테고리 배너 문구는 카드별 폭에 맞게 길이를 조정하고, 긴 단어는 줄바꿈 포인트(`<wbr>`) 및 반응형 폭 제한으로 레이아웃 깨짐을 방지한다.
  - 홈 `catHub`(모바일/태블릿)는 탭 전환 UI를 제거하고 루트 카테고리 + 하위 카테고리를 순차 스택형으로 노출해 하위 카테고리 잘림을 방지한다.
  - 데스크톱 `catStackItem` 계열 배너는 내부 텍스트/하위 요소가 잘리지 않도록 카드 외곽 여백과 최소 높이를 충분히 확보한다.
  - `catStackItem--eng` 배너는 우측 정렬로 인한 텍스트 잘림을 방지하기 위해 좌측 정렬을 유지한다.
  - 헤더 확장 GNB(`menu-footer`)는 `About → TechWrite → GuestBook → GitHub` 링크를 우선 노출하되, 티스토리 기본 동작인 `구독하기`, `로그인/로그아웃` 블록을 함께 유지한다.
  - 헤더 확장 GNB에서 카테고리 트리 목록은 숨기고, 4개 링크(`About/TechWrite/GuestBook/GitHub`)는 반응형 전 구간에서 좌측 정렬로 노출한다.
