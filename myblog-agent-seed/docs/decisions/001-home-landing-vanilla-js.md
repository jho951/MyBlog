# ADR 001: 메인 홈페이지 랜딩 허브를 바닐라 JS 데이터 렌더링으로 구성

## 상태

채택됨 (2026-02-25)

## 배경

- 티스토리 스킨 제약 때문에 React/SPA 전환 없이 `main.html` + CSS + JS 커스터마이징이 필요하다.
- 기존 메인 화면은 카테고리/구독/인용/장식 섹션이 혼합되어 있어 요구사항의 IA 순서(소개 → 탐색 → 신뢰 → 카테고리 → 최신글)를 충족하기 어렵다.
- 티스토리 최신글 치환자 블록은 이미 존재하므로, 이를 재사용하면 추가 API 없이 최신 글 목록을 구성할 수 있다.

## 결정

- `src/pages/main.html` 상단에 요구사항 중심 랜딩 허브 섹션을 추가한다.
- 허브 내부의 Quick Links / Featured Tabs / About Strengths / Tech Snapshot / Connect 카드들은 바닐라 JS 데이터 객체로 렌더링한다.
- `#ttLatestTrack`의 기존 최신글 카드 DOM을 읽어 `Recent Posts` 리스트를 재구성한다.
- 기존 카테고리 허브(`catHub`, `catStack`)와 구독 CTA(`ttSubscribePromo`)는 유지하고, 중복되거나 장식 중심 섹션은 CSS로 홈 화면에서 숨긴다.

## 영향

- 장점:
  - 티스토리 기능을 유지하면서도 요구사항 IA에 맞는 홈 랜딩 구조를 빠르게 반영할 수 있다.
  - 홈 섹션 콘텐츠를 JS 데이터 배열로 관리하여 반복 수정이 쉽다.
  - 최신 글 목록을 별도 API 없이 재구성할 수 있다.
- 단점:
  - 대표 콘텐츠/링크 데이터가 정적 설정이므로 운영자가 수동 업데이트해야 한다.
  - 최신 글 데이터는 기존 슬라이더 DOM 구조에 의존하므로 마크업 변경 시 재검토가 필요하다.

## 후속 디자인 메모 (2026-02-25)

- `theme-craft` 테마에서는 홈 섹션 래퍼에 배경/테두리를 중첩해서 주기보다, `project/cat/rp4` 내부 카드 단위의 시각 계층(그림자/입체감)으로 구획을 표현한다.
- 특히 `rp4-card`는 섹션 배경과 분리된 3D 카드 스타일을 적용해 섹션 배경 블록 없이도 클릭 가능한 정보 카드로 인지되도록 한다.
- 후속 조정으로 `rp4-card` 박스 장식은 제거하고, `rp4-icon` 자체에 입체감을 집중시켜 카드보다 아이콘 중심 정보 탐색 경험으로 전환한다.
- 카테고리 배너는 CTA 버튼 텍스트를 중복 노출하지 않고 배너 전체 클릭 affordance로 통일한다.
- 카테고리 배너 CTA 제거 이후에는 텍스트 리듬(여백/패딩)을 별도 보정해 배너 높이 대비 정보 밀도를 유지한다.
- 카테고리 배너 카피는 카드별 비대칭 레이아웃에 맞춰 길이를 다르게 관리하고, 긴 영문 단어는 `<wbr>` + CSS 폭 제한으로 반응형 안정성을 확보한다.
- 모바일/태블릿의 홈 카테고리 탐색은 탭 전환보다 스택형 노출을 우선해, 한 화면에서 루트/하위 카테고리 전체를 확인할 수 있게 한다.
- 데스크톱 비대칭 배너(`catStackItem--*`)는 카드 자체 padding/min-height를 늘려 내부 콘텐츠 클리핑 리스크를 낮춘다.
- `Engineering` 데스크톱 배너는 우하 정렬 대신 좌하 정렬로 고정해 우측 클리핑 가능성을 줄인다.
- 헤더 확장 GNB는 카테고리 기본 목록 외 보조 액션을 `About / TechWrite / GuestBook / GitHub` 링크로 단순화한다.
- 단, 티스토리의 기본 구독/인증 플로우는 유지하기 위해 `구독하기` 버튼과 `로그인/로그아웃` 블록은 GNB 하단에 함께 둔다.
- 헤더 확장 메뉴에서 카테고리 트리(`[##_category_list_##]`)는 제거하고, 핵심 4링크를 좌측 정렬된 동일 패턴 버튼으로 통일한다.
