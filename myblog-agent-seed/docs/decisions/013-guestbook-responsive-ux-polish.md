# ADR 013: guestbook 페이지 반응형/UX 보정

- 상태: 승인
- 날짜: 2026-03-01

## 배경

- 기존 guestbook은 기능 동작은 가능하지만, 페이지 헤더/가이드가 없어 맥락 전달이 약했다.
- 전역 `tt-comment-cont` 스타일(카드 회전, 입력 폼 sticky)이 모바일에서도 동일하게 적용되어 가독성과 입력 안정성이 떨어졌다.

## 결정

- guestbook 섹션에 전용 헤더(`kicker/title/description/chips`)를 추가해 페이지 목적을 명확히 한다.
- `#pageGuestbook` 범위에서 댓글 카드 간격/패딩/회전 강도를 반응형으로 보정한다.
- 입력 폼은 모바일/태블릿에서 `position: static`으로 두고, PC(1024px+)에서만 sticky를 유지한다.
- 작성 필드(이름/비밀번호/본문/등록 버튼) 크기와 배치를 모바일 우선으로 재정의한다.

## 결과

- guestbook 페이지의 첫 인지와 작성 진입 흐름이 개선된다.
- 모바일 키보드 환경에서 입력 폼 충돌/가림 이슈 가능성이 줄어든다.
- PC에서는 기존의 빠른 작성 경험(sticky composer)을 유지한다.
