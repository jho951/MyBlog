# theme-craft 홈 비주얼 정리 반복 프롬프트

1. `src/style/style.css`의 `body.theme-craft` 관련 홈 섹션(`#ttHomeLanding`) 배경/테두리 규칙을 먼저 확인합니다.
2. 섹션 래퍼 배경으로 영역을 나누기보다, 내부 카드(`project`, `rp4`, list-card 등)의 그림자/입체감으로 시각적 계층을 만듭니다.
3. 최종 오버라이드는 `src/style/style.css` 파일 끝단에 추가해 우선순위를 확보합니다.
4. `theme-craft`와 기본 테마 모두에서 텍스트 대비(특히 배너 위 흰색 텍스트)를 확인합니다.
5. `rp4` 영역은 섹션 박스보다 `rp4-icon` 입체감 중심으로 조정하고, hover 시 과도한 카드 변형이 없는지 확인합니다.
6. 변경 후 `docs/REQUIREMENTS.md`, `docs/decisions`, `prompts`를 함께 갱신합니다.
7. 카드 전체가 클릭 가능한 경우 `바로가기` 같은 중복 CTA 텍스트는 제거해 정보 밀도를 조정합니다.
8. CTA 제거 후에는 `kicker/title/desc` 간격과 카드 내부 패딩을 함께 조정해 빈 여백이 남지 않게 정리합니다.
9. 비대칭 배너 카피는 카드별 폭에 맞게 길이를 분리하고, 긴 영문 단어는 `<wbr>`/폭 제한으로 반응형 줄바꿈을 제어합니다.
10. 모바일/태블릿에서 하위 카테고리 카드가 잘리면 탭/가로 스크롤보다 스택형(또는 2열 고정 그리드)로 전환해 탐색 가시성을 우선합니다.
11. 데스크톱 `catStackItem--*`에서 잘림이 보이면 `overflow`, 카드 padding, `min-height`, 내부 텍스트 패딩을 함께 조정합니다.
12. 우측 정렬 배너에서 텍스트가 잘리면 `align-items:flex-start`, `text-align:left`, `margin-left:0`으로 즉시 되돌립니다.
13. 헤더 확장 GNB는 필요 링크만 유지하고(예: `About`, `TechWrite`, `GuestBook`, `GitHub`) 버튼/로그인 블록은 목적이 겹치면 제거합니다.
14. 다만 티스토리 기본 기능 유지 요구가 있으면 `구독하기`와 `로그인/로그아웃` 블록을 즉시 복구합니다.
15. 헤더 확장 GNB에서 카테고리 트리는 감추고, 핵심 링크 버튼은 반응형 전 구간 좌측 정렬을 기본값으로 유지합니다.
