# guestbook 반응형 UX 보정

목표:
- `guestbook` 페이지를 모바일/태블릿/PC 전 구간에서 읽기/작성 흐름이 자연스럽도록 다듬는다.
- 시각적 콘셉트(포스트잇 카드)는 유지하되 입력 폼 사용성과 가독성을 우선한다.

작업 지시:
- `src/pages/main.html`의 `<s_guest>` 영역에 페이지 목적을 설명하는 헤더 구조를 추가한다.
- `src/style/style.css`에서 `#pageGuestbook` 범위의 전용 오버라이드를 작성한다.
- 모바일에서는 입력 폼 sticky를 해제하고, PC(1024px+)에서만 sticky composer를 적용한다.
- 댓글 카드 패딩/회전/간격, 작성 필드 높이/버튼 크기를 브레이크포인트(768/1024) 기준으로 조정한다.

검증 기준:
- 모바일에서 댓글 카드/입력폼이 화면 폭을 넘지 않는다.
- 모바일 키보드 입력 시 작성 폼이 가려지지 않고 자연스럽게 스크롤된다.
- 1024px+에서 작성 폼은 우측 하단 흐름으로 sticky 동작을 유지한다.
