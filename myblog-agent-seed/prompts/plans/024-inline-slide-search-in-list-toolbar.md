# Plan Prompt 024

목표: `pageList` 상단 툴바에서 검색 버튼 클릭 시 입력창이 하단 패널이 아니라 버튼 옆에서 가로로 슬라이드 확장되도록 UI/동작을 수정한다.

요구사항:
- `list-search-panel`을 툴바 내부에 배치한다.
- `hidden` 토글이 아닌 `is-open` 클래스 기반으로 열림/닫힘 상태를 제어한다.
- CSS는 `max-width + opacity + transform` 전환으로 구현한다.
- 모바일/태블릿/데스크톱에서 확장 폭이 자연스럽게 보이도록 breakpoint별 폭을 조정한다.
- 기존 검색 필터 기능(제목/카테고리 필터링, 지우기 버튼, 결과 카운트)은 유지한다.

체크리스트:
1. 돋보기 버튼 클릭 시 input이 아이콘 우측에서 부드럽게 열리는가?
2. `ESC` 입력 시 input이 닫히고 포커스가 검색 버튼으로 복귀하는가?
3. 리스트/카드 토글 버튼 정렬이 깨지지 않는가?
4. 결과 카운트 문구(`ttListSearchMeta`)가 정상 갱신되는가?
